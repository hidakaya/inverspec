# Phase 5 — Environment and operations

## Goal
Capture how the system is configured, deployed, and operated.

## Steps
1. List environment variables and config files (without secret values).
2. Describe build, test, and deploy pipelines.
3. Note observability: logs, metrics, tracing, alerts.
4. Save `docs/spec/operations.md`.

## Output checklist
- [ ] Configuration inventory (names only for secrets)
- [ ] Deployment topology
- [ ] Runbooks or operational risks called out in code
