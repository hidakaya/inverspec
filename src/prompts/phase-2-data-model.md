# Phase 2 — Data model

## Goal
Document persistent data: schemas, entities, migrations, and validation rules.

## Steps
1. Extract or infer tables/collections, keys, and relationships.
2. Map ORM models / repositories to storage.
3. Capture validation, enums, and invariants enforced in code.
4. Save `docs/spec/data-model.md`.

## Output checklist
- [ ] Entity list with relationships
- [ ] Migration / schema sources of truth
- [ ] Validation and business invariants tied to data
