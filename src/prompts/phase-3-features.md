# Phase 3 — Feature specifications

## Goal
Describe externally visible behavior: routes, screens, APIs, and user-visible flows.

## Steps
1. Enumerate routes or endpoints with methods and handlers.
2. For each major feature, document inputs, outputs, errors, and side effects.
3. Link UI surfaces to backend endpoints where applicable.
4. Save `docs/spec/features.md` (or split by domain).

## Output checklist
- [ ] Route / endpoint catalog
- [ ] Feature-level behavior notes
- [ ] Error and edge-case behavior
